## 📋 操作步骤速查表

### 第 1-3 步：学生报名
| 步骤 | 操作 | 等待 |
|-----|------|------|
| 1 | 学生登入 `/login` | 进入 `/student_courses` |
| 2 | 访问 `/pricing` 查看点数 | 记录当前点数 |
| 3 | 购买点数（可选）：点击购买 → 结算 → 模拟支付 | 刷新页面确认 |
| 4 | 选择课程，点击"立即报名课程" | 弹出对话框 |
| 5 | 选择"点数报名"标签，**设置当前时间或未来5分钟** | - |
| 6 | 点击"确认报名" | 返回 `/student_courses` |

✅ **检查点**：点数已从原来的数值扣除了对应课程成本

### 第 4 步：确认点数扣除
| 操作 | 预期结果 |
|------|---------|
| 访问 `/pricing` | 显示新余额 = 原余额 - 课程成本 |

### 第 5 步：学生进入等待页
| 操作 | 预期结果 |
|------|---------|
| 在 `/student_courses` 点击"进入教室" | 进入 `/classroom/wait` |
| 点击"设备检测" | 允许摄像头和麦克风权限 |
| 点击"准备好" | 状态变为"已准备好"（绿色） |

### 第 6 步：教师进入等待页
| 操作 | 预期结果 |
|------|---------|
| **新浏览器/隐身窗口**登入 `/login`（lin@test.com） | 进入 `/teacher_courses` |
| 在课程列表中找到**相同的课程** | 确认 courseId 一致 |
| 点击"进入教室" | 进入 `/classroom/wait` |
| 点击"设备检测" → 允许权限 | - |
| 点击"准备好" | 状态变为"已准备好"（绿色） |

✅ **检查点**：学生页面应显示"教师已准备好"

### 第 7 步：双方进入教室
| 操作 | 预期结果 |
|------|---------|
| 学生点击"进入教室" | 进入 `/classroom/room` |
| 教师点击"进入教室" | 进入 `/classroom/room` |
| 等待 3-5 秒 | 双方能看到彼此的视频（如已开启） |

### 第 8 步：教室互动测试
| 功能 | 如何测试 |
|------|---------|
| 聊天 | 右侧 Chat 标签 → 输入消息 → 对方实时接收 |
| 白板 | 中央 Whiteboard → 绘图 → 对方实时看到 |
| 音视频 | 左侧按钮开/关 → 对方实时同步 |

## ⌨️ 浏览器快捷键
| 按键 | 功能 |
|------|------|
| **F12** | 打开开发工具（查看控制台、网络请求） |
| **F5** | 刷新页面 |
| **Ctrl+R** | 硬刷新（清除缓存后刷新） |
| **Ctrl+Shift+Delete** | 打开清除数据对话框 |
| **Ctrl+N** | 打开新标签页（用于教师登入） |
| **Ctrl+Shift+N** | 打开隐身窗口（完全隔离数据） |

## ❌ 常见问题速解

### Q: "进入教室" 按钮显示为灰色
👉 检查课程开始时间是否已过期，重新报名使用当前时间

### Q: 教师看不到学生（或反之）
👉 检查 URL 中的 `courseId` 是否相同，都刷新页面试试

### Q: 页面一直空白
👉 清除浏览器缓存（Ctrl+Shift+Delete），然后 F5 刷新

### Q: 设备检测失败
👉 尝试 HTTPS 模式：**npm run dev:https**

### Q: 点数没有扣除
👉 刷新页面或清除缓存，检查浏览器控制台（F12）是否有错误
