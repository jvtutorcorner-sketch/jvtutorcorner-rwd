# 浏览器操作指南：学生报名 → 点数确认 → 进入教室 → 开始上课

此指南用于在**实际浏览器中手动操作**完整的学生报名流程、点数确认及教室进入流程。

---

## 第一步：启动开发服务器

在项目根目录打开终端，执行以下命令之一：

### 选项 A：HTTP 模式（推荐用于测试）
```bash
npm run dev:http
# 或者
npm run dev
```
然后访问：**http://localhost:3001** 或 **http://localhost:3000**

### 选项 B：HTTPS 模式（用于生产环境测试）
```bash
npm run dev:https
```
然后访问：**https://localhost:3000**

### 选项 C：HTTPS 代理模式（可选）
```bash
npm run dev:proxy
```

## 第三步：学生报名课程流程

### 3.1 学生登录

1. **打开浏览器第一个标签**
2. 访问 **http://localhost:3000/login**（或您启动的服务地址）
3. 输入以下信息：
   - **邮箱**：`basic@test.com`
   - **密码**：`123456`
   - **验证码**：输入 `jv_secret_bypass_2024` （将自动绕过验证）
4. 点击 **登入** → 进入 `/student_courses` 页面
5. 确认右上角显示学生名称 "三 张" ✓

### 3.2 点数检查

1. 进入 `/pricing` 查看当前点数余额
   - 访问 **http://localhost:3000/pricing**
   - 观察标题出现 "X 点" 的余额显示
   - 记录当前点数（例如：0 点、10 点、100 点）

### 3.3 购买点数（如果点数不足）

如果当前点数 **小于 10 点**（示例课程需要 10 点）：

1. 在 `/pricing` 页面点击任意 **购买点数** 方案
2. 进入 `/pricing/checkout` 页面
3. 选择支付方式后点击 **模拟支付 (Demo)**
4. 等待页面刷新，系统自动返回 `/pricing` 或 `/plans`
5. 刷新页面或等待 2 秒，确认页面数字显示已更新（例如：已购买 20 点 或 100 点）

### 3.4 选择课程并报名

1. **返回课程列表**
   - 访问 **http://localhost:3000** （首页）
   - 或点击导航栏 → **课程** → 选择任意课程卡片
   
2. **进入课程详情页** `/courses/{courseId}`
   - 确认能看见课程信息（课程名、讲师、时间等）
   - 看到 **立即报名课程** 按钮

3. **点击 "立即报名课程"** 按钮
   - 弹出报名对话框
   - 选择 **点数报名** 标签（如有多个选项）
   - 系统会显示：
     - 点数成本（例如：10 点）
     - 当前余额（例如：20 点）

4. **设置课程时间**
   - 在对话框的 **开始时间** 字段，输入**当前时间** 或**未来 5 分钟**
     - 格式：`2026-03-28 16:30` （YYYY-MM-DD HH:mm）
     - 或直接使用浏览器日期选择器
   - 目的：使课程处于"进行中"状态，这样"进入教室"按钮才会显示

5. **点击 "确认报名"** 按钮
   - 系统计算点数扣除
   - 显示确认信息
   - 页面自动跳转回 `/student_courses`

---

## 第四步：确认点数扣除（课程点数页面）

### 4.1 验证点数扣除

1. **访问课程点数页面**
   - 点击导航栏 → **课程点数** 或直接访问 **http://localhost:3000/pricing**

2. **查看点数余额**
   - 观察页面顶部显示的点数（例如：原来 20 点 → 现在 10 点）
   - 确认 **已正确扣除**（原余额 - 课程成本 = 新余额）

3. **截图或记录**
   - 记录扣除前后的点数变化
   - 如果需要，可以打开浏览器开发工具观看验证

---

## 第五步：学生进入教室（等待页）

### 5.1 学生进入课程清单

1. **访问学生课程清单**
   - 点击导航栏 → **我的课程** 或直接访问 **http://localhost:3000/student_courses**

2. **找到刚才报名的课程**
   - 在课程列表中找到该课程行
   - 确认 **开始时间** 在当前时间范围内
   - 观察 **进入教室** 按钮（应该显示为蓝色可点击链接）

### 5.2 学生点击"进入教室"

1. **点击 "进入教室"** 按钮
2. **等待页面加载** → 进入 `/classroom/wait?courseId=...&orderId=...`
3. 确认页面显示：
   - 课程名称
   - 当前角色：**学生**
   - 设备检测区域（摄像头、麦克风）
   - **设备检测** 按钮

### 5.3 学生准备设备

1. **点击 "设备检测"** 按钮
   - 浏览器弹出设备权限询问
   - 点击 **允许** 或 **Allow**
   - 等待 2-3 秒，系统检测摄像头和麦克风

2. **点击 "准备好"** 按钮
   - 页面左下角出现绿色状态：**已准备好**
   - 当前页面会等待教师也进入并准备好

## 第六步：教师进入教室（等待页）

### 6.1 教师登录（新浏览器标签或隐身窗口）

1. **打开新浏览器标签或隐身/无痕窗口**（防止与学生账号混淆）

2. **访问登录页** **http://localhost:3000/login**

3. 输入教师信息：
   - **邮箱**：`lin@test.com`
   - **密码**：`123456`
   - **验证码**：`jv_secret_bypass_2024`

4. 点击 **登入** → 进入 `/teacher_courses` 页面（教师课程列表）

5. 确认右上角显示教师名称 ✓

### 6.2 教师进入同一个课程的教室

1. **在教师课程列表中找到相同课程**
   - 访问 **http://localhost:3000/teacher_courses**
   - 查找您刚才报名的课程

2. **确认课程状态**
   - 检查 **开始时间** 是否在当前时间范围内
   - 观察 **进入教室** 按钮

3. **教师点击 "进入教室"** 按钮
   - 进入 `/classroom/wait?courseId=...&orderId=...` 等待页

### 6.3 教师准备设备

1. **点击 "设备检测"** 按钮
   - 浏览器弹出设备权限询问
   - 点击 **允许** 或 **Allow**

2. **点击 "准备好"** 按钮
   - 页面左下角出现绿色状态：**已准备好**

---

## 第七步：双方进入教室（/classroom/room）

### 7.1 页面同步验证

当**学生和教师都点击了"准备好"**后：

1. **学生页面**会显示：
   - ✓ 学生已准备好
   - ✓ 教师已准备好（或显示教师在线）
   - 按钮文案可能变更为 **立即进入教室** 或 **进入教室**

2. **教师页面**会显示：
   - ✓ 教师已准备好
   - ✓ 学生已准备好（或显示学生在线）

### 7.2 双方点击"进入教室"

1. **学生操作**：点击等待页上的 **进入教室** 按钮
   - 页面跳转 → `/classroom/room?courseId=...&orderId=...`
   - 看到 Agora 音视频加载中

2. **教师操作**：同时（或之后）点击等待页上的 **进入教室** 按钮
   - 页面跳转 → `/classroom/room?courseId=...&orderId=...`

---

## 第八步：教室内操作（/classroom/room）

### 8.1 等待连接建立

进入教室后，请耐心等待 3-5 秒：

1. **Agora 连接初始化**
   - 页面加载音视频框架
   - 本地视频预览显示为自己
   - 看到对方视频（如已开启摄像头）

2. **音视频测试**
   - 左侧面板可能显示本地音视频状态
   - 可以点击 **麦克风图标** 开/关麦克风
   - 可以点击 **摄像头图标** 开/关视频

### 8.2 互动白板测试

1. **发送聊天消息**
   - 点击右侧面板的 **聊天** 或 **Chat** 标签
   - 在文本框输入消息
   - 点击 **发送** 按钮
   - 观察对方页面是否收到消息

2. **使用白板**（如需要）
   - 点击中央 **白板** 或 **Whiteboard** 标签
   - 选择绘图工具（笔、橡皮等）
   - 在白板上绘图或书写
   - 观察对方是否实时看到您的内容

### 8.3 结束课程

#### 学生操作：
1. 点击页面右上角 **离开** 或 **Leave** 按钮
2. 页面自动返回 `/student_courses`

#### 教师操作：
1. 点击页面右上角 **结束课程** 或 **End Class** 按钮
2. 弹出确认对话框
3. 点击 **确认** 或 **Confirm**
4. 页面自动返回 `/teacher_courses`

## 常见问题排查

### Q1: 登入后页面空白或一直加载中

**解决方案**：
1. 按 `Ctrl + Shift + Delete` 清除浏览器缓存
2. 关闭浏览器开发工具（按 F12）
3. 刷新页面（Ctrl + R）
4. 如果仍未解决，检查浏览器控制台（F12 → Console）是否有红色错误

### Q2: "进入教室"按钮显示为灰色或不可点击

**可能原因和解决**：
1. **课程时间不在范围内**
   - 检查您设置的开始时间是否 ≥ 当前时间
   - 如果设置为未来时间，请等到该时间或重新报名换个时间

2. **未登入学生账号**
   - 检查右上角是否显示用户名
   - 如显示"登入"，则需要重新登入

3. **点数不足**
   - 虽然已报名，但检查 `/pricing` 是否还显示点数不足信息

### Q3: 学生或教师进入等待页后，对方看不到

**可能原因和解决**：
1. **确保在同一个课程**
   - 教师课程列表中的课程 ID 应与学生报名的课程 ID 相同
   - 可以在浏览器地址栏检查 `courseId` 参数

2. **检查 localStorage**
   - 按 F12 → Application → Local Storage
   - 确保 `tutor_mock_user` 包含正确的 email 和 role

3. **刷新等待页**
   - 如果对方长时间未出现，尝试刷新（F5）

4. **检查网络连接**
   - 确保两个浏览器都能访问服务器
   - 如使用隐身窗口，某些网络配置可能不同

### Q4: "进入教室"后页面空白或视频加载不出来

**可能原因和解决**：
1. **检查浏览器权限**
   - 某些浏览器（如 Firefox）需要手动允许摄像头/麦克风权限
   - 检查浏览器地址栏左侧的权限图标

2. **Agora 初始化中**
   - 首次加载可能需要 3-5 秒
   - 请稍候，不要立即关闭

3. **检查控制台错误**
   - 按 F12 → Console
   - 查看是否有红色错误信息（如 Agora Token 过期）

### Q5: 设备检测失败（"无法访问摄像头/麦克风"）

**可能原因和解决**：
1. **HTTP vs HTTPS**
   - 浏览器在 HTTP 环境下可能限制摄像头权限
   - 尝试使用 HTTPS 地址：`npm run dev:https`

2. **权限被拒绝**
   - 检查浏览器设置 → 隐私和安全 → 网站权限
   - 确保 `localhost:3000` 被允许访问摄像头和麦克风

3. **设备占用**
   - 如果同时打开多个标签或应用使用摄像头，可能冲突
   - 关闭其他使用摄像头的应用或标签

| 6 | 报名对话框 | 设置时间 → 确认报名 | 跳转回 /student_courses |
| 7 | 确认点数扣除 | /pricing | 验证新余额正确 |
| 8 | 学生进入教室 | /student_courses → 进入教室 | 进入 /classroom/wait |
| 9 | 学生设备检测 | 准备好按钮 | 学生状态变为"已准备好" |
| 10 | 教师新窗口登入 | /login (新窗口) | 进入 /teacher_courses |
| 11 | 教师进入教室 | /teacher_courses → 进入教室 | 进入 /classroom/wait |
