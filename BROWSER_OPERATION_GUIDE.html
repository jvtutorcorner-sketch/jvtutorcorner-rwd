操作时间估算

阶段 预计时间 说明
启动服务 → 登入 3-5 分钟 首次启动较慢，包括编译时间
报名课程 2-3 分钟 包括购买点数（如需）
确认点数 → 进教室 2 分钟 快速导航确认
教师登入 → 进教室 2-3 分钟 新窗口登入
双方设备检测 → 进教室 3-5 分钟 包括浏览器权限提示
总计 12-20 分钟 首次可能更长，之后会更快

所需设备和软件

  • 电脑：Windows / Mac / Linux
  • 浏览器：Chrome / Firefox / Edge / Safari（支持 WebRTC）
  • Node.js：v16 或更高版本
  • 网络：至少 2 个独立的网络连接或同一网络下的 2 部设备
  • 硬件：摄像头和麦克风（可选，设备检测会提示）

🚀第一步：启动开发服务器

1打开终端或命令提示符

  • Windows：按 Win + R，输入 cmd 或 powershell
  • Mac：打开 Terminal（应用 → 工具程序 → 终端）
  • Linux：打开您的终端应用

2进入项目目录

cd D:\jvtutorcorner-rwd

（请根据您的实际项目路径调整）

3选择启动方式

🔹 选项 A：HTTP 模式（推荐）
npm run dev:http

访问：http://localhost:3001

🔹 选项 B：标准开发模式
npm run dev

访问：http://localhost:3000

🔹 选项 C：HTTPS 模式
npm run dev:https

访问：https://localhost:3000

✅ 包含消息 "ready - started server on 0.0.0.0:3000" 或类似消息时，服务器已成功启动

4打开浏览器访问

  • 打开任意浏览器（Chrome / Firefox / Edge）
  • 在地址栏粘贴上述链接（例如：http://localhost:3000）
  • 按 Enter 访问
⚠️ 首次启动可能需要 1-2 分钟，因为 Next.js 需要编译项目。请耐心等待。

👤第二步：准备测试账号

检查环境变量配置

1定位 .env.local 文件

  • 打开项目根目录
  • 查找名为 .env.local 的文件
  • 用文本编辑器打开（例如：VS Code、Notepad++）

2确保包含以下配置

# 学生测试账号 TEST_STUDENT_EMAIL=basic@test.com TEST_STUDENT_PASSWORD=123456 # 教师测试账号 TEST_TEACHER_EMAIL=lin@test.com TEST_TEACHER_PASSWORD=123456 # 验证码绕过密钥 LOGIN_BYPASS_SECRET=jv_secret_bypass_2024
✅ 如果缺少这些行，请复制上述代码粘贴到 .env.local 底部，然后保存。

账号信息速查表

角色 邮箱 密码 说明
学生 basic@test.com 123456 学生账号，用于报名课程
教师 lin@test.com 123456 教师账号，用于授课
验证码 jv_secret_bypass_2024 在登入页面的验证码字段输入
💡 提示：这些是测试账号。在生产环境中会有不同的账号管理方式。

📝第三步：学生报名课程

1学生登入

  1. 打开第一个浏览器标签
  2. 访问 http://localhost:3000/login
  3. 输入以下信息：
    • 邮箱：basic@test.com
    • 密码：123456
    • 验证码：jv_secret_bypass_2024
  4. 点击 登入 按钮
  5. 等待跳转 → 应进入 /student_courses 页面
✅ 右上角应显示学生名称"三 张"或类似信息

2检查点数（可选）

  1. 点击导航栏 → 课程点数
  2. 或直接访问 http://localhost:3000/pricing
  3. 观察页面显示的点数余额（例：0 点、10 点、100 点）
  4. 记录当前点数（稍后需要验证扣除）

3购买点数（如点数不足）

如果当前点数 ≥ 10，可跳过此步骤

  1. 在 /pricing 页面找到任意点数套餐
  2. 点击 购买点数 或 立即购买 按钮
  3. 进入 /pricing/checkout 结算页面
  4. 选择支付方式（例如：PayPal、信用卡等）
  5. 点击 模拟支付 (Demo) 按钮
  6. 等待 2-3 秒 → 页面自动返回 /pricing 或 /plans
  7. 刷新页面（按 F5） → 确认点数已增加
✅ 点数应从原来的点数增加（例如：0 点 → 20 点 / 100 点）

4选择课程并报名

  1. 访问首页 http://localhost:3000
  2. 或从导航栏点击 课程 → 浏览课程列表
  3. 点击任意课程卡片 → 进入课程详情页 /courses/{courseId}
  4. 找到 立即报名课程 按钮，点击

5填写报名信息

弹出的报名对话框中：

  1. 选择 点数报名 标签（如有多个选项）
  2. 页面会显示：
    • 点数成本（例如：10 点）
    • 您的余额（例如：20 点）
  3. 在 开始时间 字段，输入以下任一时间：
    • 当前时间（例如：2026-03-28 16:30）
    • 或选择 未来 5 分钟内的时间
  4. 💡 为什么需要设置当前/近期时间？因为"进入教室"按钮只在课程时间范围内显示
⚠️ 重要：如果设置开始时间为未来（超过 5 分钟），"进入教室"按钮将不会显示。请确保使用当前或很近的时间。

6确认报名

  1. 确认所有信息无误
  2. 点击 确认报名 按钮
  3. 系统处理报名请求（约 1-2 秒）
  4. 页面自动跳转回 /student_courses（我的课程）
✅ 在课程列表中应看到刚才报名的课程

💰第四步：确认点数扣除

此步骤验证点数是否正确从您的账户扣除

1访问课程点数页面

  1. 点击导航栏 → 课程点数
  2. 或直接访问 http://localhost:3000/pricing

2查看点数余额

  1. 观察页面顶部显示的点数（通常带"点"字，例如："10 点"）
  2. 对比与第二步中记录的点数：
    • 示例 1：之前 20 点 → 现在 10 点 ✅ 正确（扣除 10 点）
    • 示例 2：之前 0 点 买 20 点 → 报名后 10 点 ✅ 正确（扣除 10 点）
    • 示例 3：之前 20 点 → 现在仍是 20 点 ❌ 异常（未扣除）
✅ 预期结果：新余额 = 原余额 - 课程点数成本
⚠️ 如果点数未扣除？
  • 检查浏览器缓存：按 Ctrl+Shift+Delete 清除缓存
  • 刷新页面：按 F5 重新加载
  • 查看浏览器控制台（F12 → Console）是否有错误

3截图记录（可选）

为了文档记录，可以：

  1. 按 Ctrl+PrintScreen 截图整个屏幕
  2. 或在浏览器按 F12 → 点击设备切换图标 → 截图该面板
  3. 保存截图以备后用

👨‍🎓第五步：学生进入教室等待页

1返回学生课程列表

  1. 点击导航栏 → 我的课程
  2. 或直接访问 http://localhost:3000/student_courses

2找到报名的课程

  1. 在课程列表中找到刚才报名的课程行
  2. 确认 开始时间 在当前时间范围内
  3. 观察 进入教室 按钮（应为蓝色可点击链接）
⚠️ 如果 "进入教室" 按钮显示为灰色或 "-"，
  • 检查开始时间是否已过期
  • 可以返回课程页面重新报名，使用更晚的时间

3点击"进入教室"

  1. 点击 进入教室 按钮
  2. 页面加载 → 进入 /classroom/wait 页面
  3. 等待 2-3 秒让页面完全加载
✅ 应看到以下内容：
  • 课程名称
  • 角色标识：学生
  • 设备检测区域（摄像头、麦克风状态）
  • 设备检测 按钮
  • 准备好 按钮（可能为灰色，device check 前）

4进行设备检测

  1. 点击 设备检测 按钮
  2. 浏览器会弹出权限请求：
    • "是否允许访问您的摄像头？" → 点击 允许 或 Allow
    • "是否允许访问您的麦克风？" → 点击 允许 或 Allow
  3. 等待 2-3 秒，系统检测您的设备
  4. 准备好 按钮应变为可点击（蓝色）
⚠️ 没有摄像头/麦克风？
  • 在开发环境 (localhost) 中，某些权限可能会自动绕过
  • 您仍可点击 "准备好" 继续

5标记为准备好

  1. 点击 准备好 按钮
  2. 页面左下角会显示：已准备好（绿色状态）
  3. 页面会等待教师也进入并准备好
💡 提示：此时耐心等待教师按照第六步进行操作

👨‍🏫第六步：教师进入教室等待页

现在需要在 新浏览器窗口或隐身标签页 中以教师身份进行操作

1打开新浏览器窗口或隐身标签

  • 方法一（推荐）：按 Ctrl+N 打开新标签页
  • 方法二：按 Ctrl+Shift+N 打开隐身/无痕窗口
  • 目的：防止与学生账号混淆

2教师登入

  1. 访问登入页 http://localhost:3000/login
  2. 输入教师信息：
    • 邮箱：lin@test.com
    • 密码：123456
    • 验证码：jv_secret_bypass_2024
  3. 点击 登入 按钮
  4. 进入教师页面 /teacher_courses
✅ 右上角应显示教师名称

3找到同一个课程

  1. 在 /teacher_courses 课程列表中找到学生报名的课程
  2. 确保课程 ID 相同（可在浏览器地址栏中检查 courseId 参数）
  3. 确认 开始时间 在当前时间范围内
  4. 观察 进入教室 按钮
⚠️ 如果在教师课程列表中看不到该课程：
  • 检查课程是否已发布
  • 课程可能需要关联特定教师
  • 提供课程 ID，手动访问：http://localhost:3000/courses/{courseId}

4教师点击"进入教室"

  1. 点击 进入教室 按钮
  2. 进入 /classroom/wait 等待页
  3. 观察页面是否显示学生已在线

5教师进行设备检测

  1. 点击 设备检测 按钮
  2. 允许浏览器访问摄像头和麦克风
  3. 等待设备检测完成

6教师标记为准备好

  1. 点击 准备好 按钮
  2. 页面左下角显示：已准备好（绿色）
  3. 此时学生的页面应该也会更新，显示教师已准备好
✅ 关键时刻：一旦教师也标记为"准备好"，
  • 学生页面会显示教师已准备好
  • "进入教室"按钮应变为活动状态
  • 双方都可以点击进入教室房间

🎥第七步：双方进入教室（/classroom/room）

✅ 前提条件：学生和教师都已在等待页标记为"准备好"

1页面同步验证

学生的浏览器应显示：

  • ✓ 学生已准备好
  • ✓ 教师已准备好
  • "进入教室" 按钮 （通常为蓝色）

教师的浏览器应显示：

  • ✓ 教师已准备好
  • ✓ 学生已准备好
  • "进入教室" 按钮 （通常为蓝色）

2学生点击"进入教室"

  1. 返回学生浏览器标签页
  2. 点击等待页上的 进入教室 按钮
  3. 页面加载 → 进入 /classroom/room
  4. 等待 3-5 秒，Agora 音视频框架初始化

3教师点击"进入教室"

  1. 返回教师浏览器标签页（新窗口/隐身窗口）
  2. 点击等待页上的 进入教室 按钮
  3. 页面加载 → 进入 /classroom/room
  4. 等待 3-5 秒，Agora 音视频框架初始化

4等待连接建立

进入教室后，系统需要时间建立音视频连接：

  1. 页面加载中：可能显示 "正在连接..." 或加载动画
  2. 本地视频预览：应看到自己的摄像头画面
  3. 远端视频：2-5 秒后应看到对方的摄像头画面
  4. 音频测试：可尝试说话，对方应能听到
⚠️ 如果 3-5 秒后仍看不到对方视频，检查：
  • 浏览器控制台（F12）是否有错误
  • 网络连接是否正常
  • Agora 权限是否被允许
✅ 成功标志：
  • 两个浏览器都进入 /classroom/room 页面
  • 双方都能看到对方的视频（如已开启摄像头）
  • 可以听到对方的声音（如已开启麦克风）

4课程工具（如有）

根据您的教室实现，可能有以下功能：

  • 📋 课程计时器 - 显示剩余课程时间
  • 🖐️ 举手功能 - 学生可举手吸引教师注意

  • 📱 屏幕共享 - 共享整个屏幕或特定窗口

  2. 弹出确认对话框："确定要结束课程吗？"
  3. 点击 确认 或 Yes
  4. 页面自动返回 /teacher_courses
  5. 💡 注意：教师结束课程后，学生可能也会被强制离开

❓常见问题排查

Q1：页面加载后一直空白或加载中

可能原因：

  • Next.js 编译中
  • 浏览器缓存问题
  • 网络连接中断

解决方案：

  • 1️⃣ 等待 2-3 分钟，开发服务器第一次启动较慢
  • 2️⃣ 按 Ctrl+Shift+Delete 清除浏览器缓存（History、Cookies、Storage）
  • 3️⃣ 关闭浏览器开发工具（F12）
  • 4️⃣ 按 F5 强制刷新
  • 5️⃣ 重启浏览器或尝试其他浏览器

Q2："进入教室"按钮显示为灰色或为"-"

可能原因：

  • 课程开始时间已过期
  • 未成功登入
  • 课程状态不正确

解决方案：

  • 1️⃣ 检查您设置的课程开始时间是否 ≥ 当前时间
  • 2️⃣ 如已过期，返回课程页面重新报名，使用当前时间
  • 3️⃣ 检查浏览器右上角是否显示您的用户名
  • 4️⃣ 刷新页面重新加载课程列表

Q3：学生或教师在等待页看不到对方

可能原因：

  • 不在同一个课程
  • 页面同步延迟
  • localStorage 数据混乱

解决方案：

  • 1️⃣ 检查两个浏览器的 URL 中 courseId 是否相同
  • 2️⃣ 两个页面都按 F5 刷新
  • 3️⃣ 打开浏览器开发工具 (F12) → Application → Local Storage
    • 检查 tutor_mock_user 字段内的 email 和 role 是否正确
    • 学生应为 "basic@test.com" / "student"
    • 教师应为 "lin@test.com" / "teacher"
  • 4️⃣ 如仍未解决，清除 localStorage 后重新登入

Q4："进入教室"后页面空白或视频加载不出

可能原因：

  • Agora SDK 初始化中（正常需要 3-5 秒）
  • 摄像头/麦克风权限未被允许
  • 浏览器不支持 WebRTC

解决方案：

  • 1️⃣ 耐心等待 5-10 秒，Agora 可能在初始化
  • 2️⃣ 按 F12 打开浏览器控制台，查看是否有红色错误
  • 3️⃣ 检查浏览器地址栏左侧是否有"摄像头/麦克风"权限图标，点击允许
  • 4️⃣ 尝试其他浏览器（推荐 Chrome）

Q5：设备检测失败（"无法访问摄像头/麦克风"）

可能原因：

  • 浏览器权限被拒绝
  • HTTP vs HTTPS 限制
  • 设备被其他应用占用

解决方案：

  • 1️⃣ HTTP 场景：某些浏览器在非 HTTPS 环境下限制摄像头权限
    • 尝试 HTTPS：npm run dev:https
    • 或访问 https://localhost:3000
  • 2️⃣ 权限管理：检查浏览器设置 → 隐私和安全 → 网站权限
    • 确保 localhost:3000 被允许访问摄像头/麦克风
    • 不在"拒绝"列表中
  • 3️⃣ 设备冲突：关闭其他使用摄像头的应用或浏览器标签
  • 4️⃣ 重启设备：重启您的摄像头或计算机

Q6：登入后收不到或显示错误的账户信息

可能原因：

  • 邮箱或密码输入错误
  • .env.local 配置缺失

解决方案：

  • 1️⃣ 重新检查 .env.local 中的账号信息
  • 2️⃣ 确保验证码字段输入的是 jv_secret_bypass_2024（而不是随机数字）
  • 3️⃣ 打开浏览器控制台（F12 → Console）查看登入 API 返回的错误信息
  • 4️⃣ 尝试手动访问 `http://localhost:3000/api/login` 测试后端

Q7：点数购买后没有增加

可能原因：

  • 页面缓存未更新
  • 支付模拟失败

解决方案：

  • 1️⃣ 按 Ctrl+Shift+Delete 清除浏览器缓存
  • 2️⃣ 访问 http://localhost:3000/pricing 重新查看
  • 3️⃣ 打开浏览器开发工具 (F12) → 检查 Network 标签中 API 调用的响应
  • 4️⃣ 检查终端是否有相关错误日志

Q8：点数已扣除，但课程列表中看不到该课程

可能原因：

  • 页面缓存问题
  • API 请求失败

解决方案：

  • 1️⃣ 刷新页面（F5）
  • 2️⃣ 清除浏览器缓存后重新访问
  • 3️⃣ 打开浏览器开发工具 → Network → 查看对 `/api/orders` 的请求
  • 4️⃣ 检查响应是否包含新报名的课程数据

Q9：网络请求显示 404 或 500 错误

可能原因：

  • 后端服务器未启动
  • API 端点不存在或有 Bug

解决方案：

  • 1️⃣ 检查终端中运行的开发服务器是否仍在运行
  • 2️⃣ 打开浏览器开发工具 (F12) → Console 和 Network 标签
  • 3️⃣ 记录具体的错误码和 API 路径
  • 4️⃣ 重启开发服务器（在终端按 Ctrl+C，然后重新运行 `npm run dev:http`）
  • 5️⃣ 提交 Bug 报告，附带错误日志
