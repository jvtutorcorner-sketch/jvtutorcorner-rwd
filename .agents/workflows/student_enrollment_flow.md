# 學生選購至上課的完整操作流程

這份文件詳細記錄了學生在平台上的端到端 (End-to-End) 操作步驟：從瀏覽方案、完成付款、確認課程報名，到最終進入虛擬教室上課。

## 1. 瀏覽與選擇 (課程與方案)
1. **瀏覽課程**：學生可以在首頁或 `/courses` 頁面瀏覽所有可用的課程。點擊感興趣的課程會進入課程詳細頁面 (`/courses/[id]`)。
2. **點擊報名**：在課程詳細頁面，學生點擊「報名課程」按鈕 (`EnrollButton`)。
3. **前往定價頁面**：若學生尚未擁有符合該課程資格的「訂閱方案 (Plan)」或「點數 (Points)」，系統會引導學生前往 `/pricing` 頁面選購。
4. **選擇方案/點數**：在定價頁面，學生可以選擇合適的「訂閱方案 (例如：Basic, Pro, Elite)」或是「點數套裝」，點擊購買後進入結帳流程 (`/pricing/checkout`)。

## 2. 結帳與付款 (Checkout)
此部分的詳細驗證邏輯與測試指令請參閱 [Point Purchase Flow Skill](../skills/point-purchase-flow/SKILL.md)。

1. **結帳頁面**：進入結帳畫面 (`/pricing/checkout?plan={planId}`)，系統會顯示訂單摘要與金額。
2. **選擇金流服務**：平台支援多種付款方式，學生可選擇：
   - **模擬支付 (Demo)**：用於測試與開發環境，即時完成訂單。
   - **信用卡 (Stripe)**：真實金流，需輸入卡號。
   - **PayPal**：真實金流，跳轉至 PayPal 登入頁。
   - **LINE Pay**：真實金流，跳轉至 LINE Pay。
3. **完成付款**：
   - **模擬模式**：直接點擊「模擬支付」，訂單立即轉為 `PAID`。
   - **真實模式**：前往第三方金流頁面完成授權。付款成功後，會被導向平台的「我的課程」(`/student_courses`) 頁面。


## 3. 系統訂單與權限更新 (後台處理)
1. **建立訂單與報名紀錄**：付款成功後，後端 API (`/api/orders`, `/api/orders/[orderId]`) 會確認金流狀態，將訂單狀態更新為 `PAID`。
2. **生成 Enrollment 紀錄**：如果是有針對特定課程報名，系統會在 `jvtutorcorner-enrollments` 資料表建立或將狀態更新為 `ACTIVE`/`PAID`。
3. **更新學生身分**：學生的帳號資料 (例如 `user.plan` 或剩餘點數) 會同步更新，取得參與課程的權限。

## 4. 檢視我的課程 (Student Courses)
1. **進入學生專區**：學生導航至 `/student_courses`（訂單紀錄 / 我的課程）。
2. **查看課程清單**：此頁面會列出學生所有已付款 (`PAID`) 的課程訂單。
3. **課堂資訊**：列表中會顯示授課老師、課程時間、**剩餘課程數**以及**剩餘時間 (分鐘)**。
4. **進入教室按鈕**：只要學生擁有有效方案（basic, pro, elite），針對該課程的唯讀行末端會顯示「**進入教室**」 (`enter_classroom`) 按鈕。

## 5. 課前準備與設備測試 (Wait Room)
1. **進入等待室**：點擊「進入教室」後，學生會被導向 `/classroom/wait?courseId={courseId}&orderId={orderId}`。
2. **硬體測試**：頁面會初始化 Agora 視訊與語音 SDK，並要求學生測試麥克風與攝影機。
3. **設定確認**：系統可能同時處理設備資訊紀錄 (Device Info) 與環境檢查。
4. **加入會議**：確認設施設備無誤後，學生點擊「加入會議」，正式進入虛擬教室。

## 6. 開始上課 (Classroom)
1. **虛擬教室**：學生進入 `/classroom/room?courseId={courseId}&orderId={orderId}`。
2. **上課功能**：在教室內，學生可以使用：
   - 即時視訊與語音通話 (Agora)
   - 互動白板
   - 聊天室與即時 AI 助理
3. **課程結束**：下課後，系統會更新該次的上課紀錄 (`COURSE_RECORDS`)，並扣除相應的剩餘堂數或時間。
